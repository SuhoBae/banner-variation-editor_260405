# 자동 루프 운영 규칙

## 목적
- 이 프로젝트를 다음 세션이나 다른 기기에서도 같은 품질로 이어서 완성할 수 있도록, 작업 루프를 문서화한다.

## 루프 기본 규칙
1. `AGENTS.md`에서 최근 Learning Log를 읽는다.
2. `실행계획.md`에서 가장 위의 미완료 항목 1개만 선택한다.
3. 선택한 항목을 구현하기 전, 관련 UX 리스크와 기존 충돌 가능성을 먼저 점검한다.
4. 구현한다.
5. `npm run build`로 반드시 검증한다.
6. 새로 배운 점, 고친 이유, 남은 리스크를 MD에 기록한다.
7. commit/push 해서 Vercel preview에 반영한다.
8. preview에서 핵심 시나리오를 확인한다.

## 루프당 산출물
- 코드 변경
- `AGENTS.md` Learning Log 갱신
- `실행계획.md` 체크리스트 갱신
- 필요 시 중간정리 문서 갱신
- GitHub push

## 우선순위 규칙
- 1순위: 편집기 핵심 UX 안정화
- 2순위: export 결과와 화면 일치
- 3순위: 에셋 라이브러리/보드별 변형 고도화
- 4순위: 구조 개선과 분리 리팩터링

## 검증 체크리스트
- 텍스트 편집: 한글 IME, Enter, Escape, 바깥 클릭
- 이미지 편집: 업로드, 크롭, 스케일, 교체
- 선택/변형: 이동, 리사이즈, 더블탭 맞춤, 보드별 독립성
- 보드별 오버라이드: 텍스트/CTA/숨김/초기화
- export: PNG 정상 생성과 화면 유사성

## 멈추면 안 되는 상황
- 구현만 하고 문서 업데이트를 생략하는 것
- build 검증 없이 push하는 것
- 현재 브랜치 상태를 기록하지 않는 것
- 원인 분석 없이 UX를 눈대중으로 반복 수정하는 것

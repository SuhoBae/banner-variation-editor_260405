# AdCanvas 중간 정리

## 현재 상태
- Vite 기반 실행 구조가 복구되어 `npm run dev`, `npm run build`가 가능한 상태다.
- GitHub 저장소와 Vercel preview가 연결되어 있어 브랜치 push만으로 다른 기기와 프리뷰 환경을 이어갈 수 있다.
- 핵심 편집기 기능은 단일 파일 [`App.jsx`](C:\Users\okiku\Downloads\260405_banner_variation\App.jsx)에 집중되어 있다.
- 보드별 오버라이드, 히스토리, 에셋 트레이, export, 인플레이스 텍스트 편집, 이미지 교체, 선택/변형 UI가 동작한다.

## 이번까지 개선된 것
- 포인터 기준 줌, 향상된 패닝/리사이즈 감도
- 보드별 레이어 오버라이드와 보드 변형 표시/초기화
- 하단 에셋 트레이와 로컬/URL 이미지 불러오기
- 다크 모드 유지
- 텍스트 인플레이스 편집과 한글 IME 조합 입력 안정화
- 이미지 체크보드 배경 제거
- 선택 라벨 표시, 사각형 핸들, 더블탭 콘텐츠 맞춤
- 선택 해제 규칙을 바깥 클릭, 탭, `Escape` 기준으로 통일

## 배운 것
- `contentEditable`는 IME 조합 입력과 React 상태 업데이트가 쉽게 충돌하므로, 입력 중에는 DOM/ref에만 유지하고 종료 시점에만 상태를 커밋하는 편이 안전하다.
- 선택 박스 크기를 레이아웃 구역 기준으로 두면 실제 요소보다 과하게 커 보이므로, 텍스트와 CTA는 실측 콘텐츠 기준 기본 박스를 써야 한다.
- 드래그 오버레이는 편집 영역과 쉽게 충돌하므로, 편집 중일 때는 오버레이를 끄거나 범위를 분리해야 한다.
- 이미지 레이어에서 체크보드 배경은 투명도 디버깅에는 유용하지만, 실제 편집 UX에서는 혼동을 만든다.
- 문서 업데이트를 루프의 일부로 강제해야 다음 세션에서 재학습 비용이 낮다.

## 자가발전에 필요한 것
- 단일 파일 구조를 모듈화해서 툴바, 캔버스, 우측 패널, 에셋 트레이, 상태 계산 로직을 분리할 필요가 있다.
- 현재 선택/편집/드래그 상태 관리가 컴포넌트 내부 로컬 상태에 흩어져 있어, 편집기 상태 머신 수준의 정리가 필요하다.
- 실측 기반 박스 계산과 export 렌더링 규칙을 공통 유틸로 합쳐 화면과 export 결과 차이를 더 줄여야 한다.
- 이미지 레이어와 텍스트/CTA 레이어의 상호작용 규칙을 더 일관되게 만들어야 한다.
- 브라우저 자동 점검 루프를 붙여 회귀를 더 빨리 잡을 필요가 있다.

## 아직 완성해야 할 것

- export 품질 점검: 멀티라인, CTA 패딩, 이미지 크롭, 작은 배너
- 문서 체계 정리: `specs/` 부재 문제 해결, 실제 기준 문서 확정
